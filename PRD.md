## 1. 專案概述

一個以 Web 呈現的手沖咖啡沖煮紀錄系統，讓使用者記錄每一次手沖的完整參數（豆子資訊、器材、沖煮變因、感官評分），並透過**桌面端的列表、篩選與趨勢圖表**，回頭分析「哪些變因造就了好喝的一杯」，進而優化沖煮。

### 1.1 產品目標
1. **完整記錄**：涵蓋一杯手沖從豆子到杯中的關鍵參數，且輸入不繁瑣。
2. **可分析**：所有影響風味的變因與感官評分皆為可量化 / 可篩選的結構化資料，支援趨勢分析與同支豆比較。
3. **可重現**：紀錄足以讓使用者重現一次成功的沖煮，並記錄「下次調整方向」形成迭代。
4. **可成長**：先上基礎版，架構預留進階量化（TDS/萃取率/水質/注水分段）與社群分享的擴充空間。

### 1.2 成功衡量指標（初期，非嚴格 KPI）
- 使用者願意在每次沖煮後完成紀錄（輸入摩擦低）。
- 能透過分析頁找出「同支豆的最佳沖法」與「養豆適飲窗口」。
- 資料結構穩定，Phase 2 擴充欄位時不需破壞性遷移。

| 角色 | 說明 | Phase |

## 3. 使用情境（User Scenarios）

- **S1 記錄一次沖煮**：使用者沖完一杯，選擇（或新增）這包豆子，填入器材與沖煮變因、感官評分與風味標籤，儲存。系統自動算出養豆天數與水粉比。
- **S2 快速再沖同一杯**：以上一筆紀錄為範本「複製」，只微調刻度/水溫等少數欄位。
- **S3 事後分析（主要情境）**：在桌面端打開某支豆的紀錄列表，比較不同刻度/水溫/粉水比下的喜好度，找出最佳沖法。
- **S4 找適飲窗口**：檢視某支豆「養豆天數 vs 整體喜好度」曲線，判斷這支豆何時最好喝。
- **S5 回顧風味偏好**：檢視風味標籤的出現頻率與平均喜好度，了解自己偏好的風味走向。
- **S6（Phase 2）分享**：將一支豆或一次沖煮設為公開，分享連結給朋友參考配方。

## 5. 功能需求（Functional Requirements）

### FR-1 帳號與認證
- FR-1.1 使用者可用 Email + 密碼（或第三方 OAuth，技術選型後決定）註冊 / 登入。
- FR-1.2 每位使用者有一個**使用者名稱（display name）**，作為沖煮紀錄的「沖煮人」自動帶入。
- FR-1.3 資料預設私有：使用者只能看見 / 編輯自己的豆子與沖煮紀錄。
- FR-1.4（Phase 2）可將特定資源設為公開分享。
- FR-1.5 使用者名稱（username）**全站唯一**，不可重複。

### FR-2 豆子管理（Bean）
- FR-2.1 建立 / 檢視 / 編輯 / 刪除豆子（一包豆子為一筆）。
- FR-2.2 焙度為固定選項：`淺焙 / 中淺焙 / 中焙 / 中深焙 / 深焙`。
- FR-2.3 刪除豆子**僅限建立者**；刪除時需「**輸入該豆名以確認**」以防誤刪。確認後採硬刪除，且其下所有沖煮紀錄一併刪除（cascade），對話框需顯示將連帶刪除的沖煮筆數。
- FR-2.4 新增沖煮紀錄時可快速新增豆子（inline 建立），減少切換。

### FR-3 沖煮紀錄（Brew）
- FR-3.1 建立 / 檢視 / 編輯 / 刪除沖煮紀錄。
- FR-3.2 每筆沖煮必須關聯一包豆子。
- FR-3.3 「沖煮人」自動帶入目前登入者的使用者名稱（FR-1.2）。
- FR-3.4 日期時間預設為當下，可修改。
- FR-3.5 感官評分（香氣 / 酸質 / 甜感 / 苦味 / 口感醇厚度 / 平衡感 / 餘韻 / 整體喜好度）皆為 1–5 整數。
- FR-3.6 可將既有紀錄「複製為新紀錄」作為範本（S2）。
- FR-3.7 冰量為選填；水粉比可選擇是否計入冰量（FR-4.2）。
- FR-3.8 每筆沖煮有**沖煮類型**（`手沖 / 冰手沖`，預設手沖）；架構支援未來新增沖煮法（如義式），不同類型未來可有不同欄位集。
- FR-3.9 「刻度」需**綁定所選磨豆機**（各機研磨度不同）；分析研磨度時以同一台磨豆機為比較範圍。
- FR-3.10 磨豆機為使用者可維護的清單（於設定管理），沖煮時從中選取。

### FR-4 自動計算
- FR-4.1 **養豆天數** = 沖煮日期 − 烘焙日期（以天為單位，唯讀顯示）。
- FR-4.2 **水粉比**：顯示為 `1:NN` 格式（NN = 水量 ÷ 粉量，四捨五入或一位小數）。
  - 提供「是否計入冰量」開關；開啟時分子為（水量＋冰量）。
- FR-4.3 計算欄位為衍生值，不儲存為可編輯欄位（或儲存但不可手動改）。

### FR-5 風味標籤系統
- FR-5.1 系統內建一批常見風味標籤（依風味輪分類，見 §8）。
- FR-5.2 使用者可為一筆沖煮選擇多個標籤（多選）。
- FR-5.3 使用者可**自行新增**標籤；新增的標籤進入其可重複使用的標籤庫。
- FR-5.4 另提供「風味自由補充」文字欄位，補充標籤未涵蓋的描述。
- FR-5.5 標籤需可用於分析（出現頻率、對應平均喜好度）。
- FR-5.6（2026-07-14 改版）使用者自訂標籤為**個人私有**；另可「**提交到群組**」：
  由**群組建立者**於設定頁審核，核可後成為 scope=group 的**群組標籤**（全體成員
  可選用）、退回則提交者仍保有個人標籤。群組建立者可直接建立群組標籤免審。
  （原「提交給系統內建、管理者審核」流程廢止——本產品無全域管理員角色。）

### FR-6 分析與趨勢（詳見 §9）
- FR-6.1 同一包豆的所有沖煮並列比較。
- FR-6.2 喜好度 vs 變因散點圖（刻度 / 水溫 / 粉水比 / 水量）。
- FR-6.3 養豆天數 vs 整體喜好度曲線。
- FR-6.4 風味標籤頻率 / 平均喜好度統計。

### FR-7 篩選與搜尋
- FR-7.1 沖煮列表可依下列條件篩選：豆子、產地、處理法、焙度、烘豆店家、濾杯型號、日期區間、喜好度區間、風味標籤。
- FR-7.2 可依日期、整體喜好度等欄位排序。
- FR-7.3 關鍵字搜尋（豆名 / 店家 / 備註）。

### FR-8 資料匯出
- FR-8.1 使用者可將自己的沖煮紀錄匯出為 CSV。

### FR-9 分享（✅ 已實作，2026-07-16 / W4；FR-9.4 配方卡匯出圖片為選配未做）
- FR-9.1 可將某支豆 / 某次沖煮設為公開，產生分享連結供他人唯讀檢視（不需登入）。
- FR-9.2（Phase 3 擴充）公開沖煮頁以「**配方卡**」形式呈現：豆子履歷、器材、變因、注水分段時間軸、評分；隱藏擁有者不想公開的欄位（自由備註、下次調整方向預設不公開）。
- FR-9.3 取消公開即失效（slug 作廢）；重新公開產生新連結。
- FR-9.4（延伸）配方卡可匯出為圖片（社群貼文用），列為 P2 內的選配項。

### FR-10 群組共豆（✅ 已實作，2026-07-14 上線）

讓一群朋友共有同一包豆子，各自記錄沖煮並互相檢視比較（「同支豆、不同人，誰沖得最好」）。

- FR-10.1 **群組**：使用者可建立群組；一人可屬多個群組。
- FR-10.2 **邀請機制＝邀請碼**：建立者產生邀請碼，朋友輸入邀請碼**申請加入**、由建立者核可（FR-10.10；不做 username 搜尋邀請，避免騷擾疑慮）。邀請碼可重新產生使舊碼失效。
- FR-10.3 **群組權限**：
  - 群組**僅建立者可刪除**（解散）；成員可自行退出。
  - 群組豆的編輯 / 刪除僅限**豆子建立者**。
- FR-10.4 **豆子歸屬**：建立豆子時可選「個人」或「某群組」；群組豆對全體成員可見，成員皆可對其新增沖煮。
- FR-10.5 **沖煮紀錄權限**：群組豆下的沖煮紀錄對全體成員**可看不可改**（僅本人可編輯 / 刪除自己的紀錄）；列表與豆子詳情顯示「沖煮人」。
- FR-10.6 **風味標籤可見性**：他人沖煮上掛的自訂標籤，對「看得到該筆沖煮的成員」開放**唯讀**（僅名稱），標籤本身仍屬原作者私有。
- FR-10.7 **分析**：群組豆相關圖表可切換「只看我的 / 全群組」。
- FR-10.8 資料模型：`groups`、`group_members`（含邀請碼）、`beans.group_id`（nullable，null＝個人豆）；RLS 政策由「僅本人」擴為「本人或同群組成員（讀）」。歷史個人資料不受影響。
- FR-10.9（2026-07-14 新增，07-15 改為審核制）**群組共用器材與磨豆機**：於「群組」頁管理（設定頁的磨豆機／我的器材為純個人）。成員新增群組器材＝**提案（pending）**，經**群組建立者核可**後全體成員才可於沖煮表單選用；建立者自己新增直接生效。pending 提案人可撤回、建立者可退回；**核可後交由建立者管理**（編輯/刪除，防提案後改名繞審）。沖煮表單的器材下拉只列「已登錄且已核可」的器材，並依豆子歸屬過濾：個人豆→只列我的個人器材；群組豆→該群組器材＋我的個人器材（靜態預設清單僅作為新增器材時的建議，不再直接出現在下拉）。刻度分析仍以「單一磨豆機」為比較範圍（FR-3.9），不跨機比較。
- FR-10.10（2026-07-15 新增）**入群審核**：輸入邀請碼改為**送出申請**（`group_join_requests`），經群組建立者**核可**後才正式入群；退回即刪除申請（可重新申請）、申請人可自行撤回。申請中僅可見群組名稱（豆子/沖煮仍不可見）。
- FR-10.11（2026-07-15 新增）**「群組」獨立分頁**：左側欄新增「群組」。列表頁顯示已加入群組（身分徽章、成員數、群組豆數、管理者的待審核計數）與申請中清單；內頁含成員（含入群審核）、邀請碼、共用器材、群組豆（可新增豆子並預選群組、一鍵沖煮、**每支豆「誰沖得最好」排行** — 各成員最高整體評分前 3 名）、群組標籤、最近沖煮動態。群組管理自設定頁全數遷出，設定頁僅保留個人設定。
- FR-10.12（2026-07-15 新增）**副群組長**：建立者可於成員名單指派/解除副組長（`group_members.role`）。副組長可審核**入群申請、器材提案、標籤提交**（含直建群組標籤/器材免審）；**仍限建立者**：解散群組、更換邀請碼、移除成員、指派副組長、管理已生效項目（刪群組標籤、編輯/刪除已核可器材）。權限判斷統一走 DB 函式 `is_group_manager()`。

- FR-13.5（✅ 已實作 2026-07-16；源自 2026-07-15 手機實測回饋）**專注計時模式**（手機 UX 修正）：
  目前計時器與注水分段的可編輯輸入列在同一張卡片，手機上每按一次「分段」

    （正好符合 FR-13.1「沖完後補填」的意圖）。

  收合成唯讀清單、結束後才展開輸入。以全螢幕模式為優先（更貼近吧台單手操作）。
  實作註記（2026-07-16）：採全螢幕方案；以 fixed 覆蓋層實作而非 Dialog

## 6. 資料模型（概念層）

三個核心實體 + 標籤關聯：

```
User (使用者)
  ├─< Grinder (磨豆機)
  ├─< Bean (豆子，一包)
  │     └─< Brew (單次沖煮) >── Grinder（刻度綁定）
  ├─< FlavorTag (自訂標籤，個人私有)
  └─< TagSuggestion (提交待審標籤)
Brew >──< FlavorTag (風味標籤，多對多)
FlavorTag ─ 系統預設（scope=system）或 User 自訂（scope=user）
（濾杯 / 濾紙 / 手沖壺：Phase 1 先以文字 / 下拉字串記錄，Phase 2 可再正規化）
```

- **User**：帳號、使用者名稱（全站唯一）。
- **Grinder**：磨豆機（屬於某 User）；沖煮的「刻度」綁定此實體。
- **Bean**：一包豆子的履歷屬性（屬於某 User，建立者）。
- **Brew**：單次沖煮的變因與評分（屬於某 User，關聯一個 Bean、可選一台 Grinder）。
- **FlavorTag**：風味標籤（系統預設或使用者自訂）；與 Brew 多對多。
- **TagSuggestion**：使用者提交、待審核成為內建的標籤。

> 設計原則：把「一包豆子會沖很多次」的重複資訊（店家 / 豆名 / 產地 / 處理法 / 品種 / 焙度 / 烘焙日期…）正規化到 Bean，避免每次沖煮重複輸入，並支撐「同支豆比較」。

## 12.5 實作狀態與已知差異（2026-07-14 回寫）

Phase 1（FR-1～FR-8）與 FR-10/11/12 全數上線。與本文件原始描述的**有意識差異**：

| 差異 | 說明 |
|---|---|
| 喜好度篩選 | FR-7.1 原寫「區間」，實作為「≥N」門檻（M3-PLAN D14） |
| 時間輸入 | 悶蒸/總時間為「分/秒下拉」（實測回饋迭代；仍存秒、顯示 m:ss） |
| users 表 | 依 Supabase 慣例更名 profiles、不存 email（M3-PLAN D4） |
| 器材 | 以「使用者/群組維護的選項清單」實現；沖煮紀錄仍存文字，非 FK |
| 列表上限 | 列表/分析分批抓取，防呆上限 5000 筆（個人量級無虞） |

已知限制：
- 退出群組後，自己記在群組豆上的沖煮會暫時從列表消失（資料仍在，重新入群即恢復）。
- 解散群組時群組標籤一併刪除（掛過該標籤的沖煮失去該標籤）；群組豆回歸建立者個人豆。
- Supabase Free 無自動備份；以 CSV 匯出作手動備份。
