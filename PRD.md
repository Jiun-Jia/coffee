## 1. 專案概述

一個以 Web 呈現的手沖咖啡沖煮紀錄系統，讓使用者記錄每一次手沖的完整參數（豆子資訊、器材、沖煮變因、感官評分），並透過**桌面端的列表、篩選與趨勢圖表**，回頭分析「哪些變因造就了好喝的一杯」，進而優化沖煮。

### 1.1 產品目標
1. **完整記錄**：涵蓋一杯手沖從豆子到杯中的關鍵參數，且輸入不繁瑣。
2. **可分析**：所有影響風味的變因與感官評分皆為可量化 / 可篩選的結構化資料，支援趨勢分析與同支豆比較。
3. **可重現**：紀錄足以讓使用者重現一次成功的沖煮，並記錄「下次調整方向」形成迭代。
4. **可成長**：先上基礎版，架構預留進階量化（TDS/萃取率/水質/注水分段）與社群分享的擴充空間。

### 1.2 成功衡量指標（初期，非嚴格 KPI）
- 使用者願意在每次沖煮後完成紀錄（輸入摩擦低）。
- 能透過分析頁找出「同支豆的最佳沖法」與「養豆適飲窗口」。
- 資料結構穩定，Phase 2 擴充欄位時不需破壞性遷移。

## 2. 目標使用者與角色

| 角色 | 說明 | Phase |
|---|---|---|
| 一般使用者（Brewer） | 註冊帳號、記錄自己的豆子與沖煮、檢視自己的分析 | Phase 1 |
| 受邀共用者 | 朋友各自有帳號，各自紀錄；資料預設私有 | Phase 1 |
| 分享 / 瀏覽者 | 未來可將某些豆子或沖煮紀錄設為公開分享給他人檢視 | Phase 2 |

> 目前定位為「多人各自紀錄、資料預設私有」，未來擴充「分享」。系統一開始即需要帳號與認證。

## 3. 使用情境（User Scenarios）

- **S1 記錄一次沖煮**：使用者沖完一杯，選擇（或新增）這包豆子，填入器材與沖煮變因、感官評分與風味標籤，儲存。系統自動算出養豆天數與水粉比。
- **S2 快速再沖同一杯**：以上一筆紀錄為範本「複製」，只微調刻度/水溫等少數欄位。
- **S3 事後分析（主要情境）**：在桌面端打開某支豆的紀錄列表，比較不同刻度/水溫/粉水比下的喜好度，找出最佳沖法。
- **S4 找適飲窗口**：檢視某支豆「養豆天數 vs 整體喜好度」曲線，判斷這支豆何時最好喝。
- **S5 回顧風味偏好**：檢視風味標籤的出現頻率與平均喜好度，了解自己偏好的風味走向。
- **S6（Phase 2）分享**：將一支豆或一次沖煮設為公開，分享連結給朋友參考配方。

## 4. 範圍（Scope）

### 4.1 Phase 1 — 基礎版（MVP）
- 帳號註冊 / 登入 / 登出。
- 豆子（Bean）CRUD。
- 沖煮紀錄（Brew）CRUD，含自動計算（養豆天數、水粉比）。
- 風味標籤系統（內建預設 + 使用者自訂）。
- 沖煮紀錄列表 + 多條件篩選 + 排序。
- 基礎分析 / 趨勢圖表（詳見 §9）。
- 複製既有紀錄為新紀錄的範本。
- 桌面優先的響應式介面（手機可用即可）。
- 資料匯出（CSV）。

### 4.2 Phase 2 — 擴充（先在架構預留，暫不實作）
- 進階量化欄位：TDS 濃度、萃取率（自動）、杯中液重、水質配方、Agtron 焙度值、室溫濕度。
- **結構化注水分段時間軸**（每段時間點＋累積水量＋手法）。
- 照片上傳（豆袋標籤 / 成品）。
- 分享 / 公開檢視、公開配方連結。
- 標籤庫的共用 / 社群化。

### 4.3 非目標（Out of Scope）
- 電商 / 購買豆子。
- 沖煮教學內容 / 影片。
- 行動原生 App（以 Web / 響應式為主，未來可評估 PWA）。
- 濃縮咖啡（Espresso）專屬參數（本版聚焦手沖 pour-over；架構不排斥未來擴充沖煮類型）。

## 5. 功能需求（Functional Requirements）

### FR-1 帳號與認證
- FR-1.1 使用者可用 Email + 密碼（或第三方 OAuth，技術選型後決定）註冊 / 登入。
- FR-1.2 每位使用者有一個**使用者名稱（display name）**，作為沖煮紀錄的「沖煮人」自動帶入。
- FR-1.3 資料預設私有：使用者只能看見 / 編輯自己的豆子與沖煮紀錄。
- FR-1.4（Phase 2）可將特定資源設為公開分享。

### FR-2 豆子管理（Bean）
- FR-2.1 建立 / 檢視 / 編輯 / 刪除豆子（一包豆子為一筆）。
- FR-2.2 焙度為固定選項：`淺焙 / 中淺焙 / 中焙 / 中深焙 / 深焙`。
- FR-2.3 刪除豆子時，若其下已有沖煮紀錄，需提示（採軟刪除或阻擋，見開放問題 Q3）。
- FR-2.4 新增沖煮紀錄時可快速新增豆子（inline 建立），減少切換。

### FR-3 沖煮紀錄（Brew）
- FR-3.1 建立 / 檢視 / 編輯 / 刪除沖煮紀錄。
- FR-3.2 每筆沖煮必須關聯一包豆子。
- FR-3.3 「沖煮人」自動帶入目前登入者的使用者名稱（FR-1.2）。
- FR-3.4 日期時間預設為當下，可修改。
- FR-3.5 感官評分（香氣 / 酸質 / 甜感 / 苦味 / 口感醇厚度 / 平衡感 / 餘韻 / 整體喜好度）皆為 1–5 整數。
- FR-3.6 可將既有紀錄「複製為新紀錄」作為範本（S2）。
- FR-3.7 冰量為選填；水粉比可選擇是否計入冰量（FR-4.2）。

### FR-4 自動計算
- FR-4.1 **養豆天數** = 沖煮日期 − 烘焙日期（以天為單位，唯讀顯示）。
- FR-4.2 **水粉比**：顯示為 `1:NN` 格式（NN = 水量 ÷ 粉量，四捨五入或一位小數）。
  - 提供「是否計入冰量」開關；開啟時分子為（水量＋冰量）。
- FR-4.3 計算欄位為衍生值，不儲存為可編輯欄位（或儲存但不可手動改）。

### FR-5 風味標籤系統
- FR-5.1 系統內建一批常見風味標籤（依風味輪分類，見 §8）。
- FR-5.2 使用者可為一筆沖煮選擇多個標籤（多選）。
- FR-5.3 使用者可**自行新增**標籤；新增的標籤進入其可重複使用的標籤庫。
- FR-5.4 另提供「風味自由補充」文字欄位，補充標籤未涵蓋的描述。
- FR-5.5 標籤需可用於分析（出現頻率、對應平均喜好度）。

### FR-6 分析與趨勢（詳見 §9）
- FR-6.1 同一包豆的所有沖煮並列比較。
- FR-6.2 喜好度 vs 變因散點圖（刻度 / 水溫 / 粉水比 / 水量）。
- FR-6.3 養豆天數 vs 整體喜好度曲線。
- FR-6.4 風味標籤頻率 / 平均喜好度統計。

### FR-7 篩選與搜尋
- FR-7.1 沖煮列表可依下列條件篩選：豆子、產地、處理法、焙度、烘豆店家、濾杯型號、日期區間、喜好度區間、風味標籤。
- FR-7.2 可依日期、整體喜好度等欄位排序。
- FR-7.3 關鍵字搜尋（豆名 / 店家 / 備註）。

### FR-8 資料匯出
- FR-8.1 使用者可將自己的沖煮紀錄匯出為 CSV。

### FR-9 分享（Phase 2）
- FR-9.1 可將某支豆 / 某次沖煮設為公開，產生分享連結供他人唯讀檢視。

## 6. 資料模型（概念層）

三個核心實體 + 標籤關聯：

```
User (使用者)
  └─< Bean (豆子，一包)
        └─< Brew (單次沖煮)
Brew >──< FlavorTag (風味標籤，多對多)
FlavorTag ─ 屬於系統預設或某 User 自訂
（器材如濾杯/磨豆機/手沖壺：Phase 1 先以文字/下拉字串記錄，Phase 2 可正規化為 Equipment 實體）
```

- **User**：帳號、使用者名稱。
- **Bean**：一包豆子的履歷屬性（屬於某 User）。
- **Brew**：單次沖煮的變因與評分（屬於某 User，關聯一個 Bean）。
- **FlavorTag**：風味標籤（系統預設或使用者自訂）；與 Brew 多對多。

> 設計原則：把「一包豆子會沖很多次」的重複資訊（店家 / 豆名 / 產地 / 處理法 / 品種 / 焙度 / 烘焙日期…）正規化到 Bean，避免每次沖煮重複輸入，並支撐「同支豆比較」。

## 9. 分析功能明細

| 分析 | 說明 | 圖表型式 |
|---|---|---|
| A1 同支豆比較 | 選定一包豆，列出其所有沖煮，可並列關鍵變因與評分 | 表格 + 標記最佳喜好度 |
| A2 喜好度 vs 變因 | X 軸為刻度 / 水溫 / 粉水比 / 水量，Y 軸為整體喜好度 | 散點圖（可切換變因） |
| A3 適飲窗口 | X 軸養豆天數，Y 軸整體喜好度 | 折線 / 散點 |
| A4 風味標籤統計 | 各標籤出現次數與對應平均喜好度 | 長條圖 / 排行 |
| A5 感官雷達 | 單筆或多筆沖煮的八維感官比較 | 雷達圖 |

- 分析可套用 §7 的篩選條件（如僅看某產地 / 某焙度）。

## 10. 非功能需求（NFR）

| 類別 | 需求 |
|---|---|
| 平台 | Web，**桌面優先**的響應式設計；手機至少可完成紀錄與檢視 |
| 瀏覽器 | 近兩版的 Chrome / Edge / Safari |
| 效能 | 一般個人資料量（數百至數千筆）下，列表與分析頁 < 1s 回應 |
| 隱私與權限 | 資料預設私有，使用者僅能存取自己的資料（後端層強制，非僅前端隱藏） |
| 認證安全 | 密碼雜湊、Session/Token 安全處理（依技術選型） |
| 資料保存 | 雲端資料庫，具備備份機制；提供 CSV 匯出 |
| 在地化 | 介面繁體中文；金額 / 單位使用公制（g、℃） |
| 可維護性 | Schema 預留 Phase 2 欄位，避免破壞性遷移 |

## 13. 開放問題 / 待確認

- **Q1** 使用者名稱是否可與他人重複？（Phase 2 分享時建議唯一或加識別碼）
- **Q2** 「刻度」是否需要綁定「磨豆機」以利跨機比較？（初版先各自文字）
- **Q3** 刪除仍有沖煮紀錄的豆子時，採「阻擋」或「軟刪除 / 封存」？
- **Q4** 悶蒸時間 / 總時間的輸入格式：純秒數 vs `m:ss`？（建議統一存秒數，顯示為 `m:ss`）
- **Q5** 風味標籤自訂是「個人私有」或「全站共用」？（初版建議個人私有，Phase 2 再共用）
- **Q6** 是否需要沖煮類型欄位（手沖 / 冰手沖 / 其他）以利未來擴充？
- **Q7** 內建風味標籤最終清單是否照 §8？
